# 实验一：安装课程所需软件实验报告

## 一、实验目的

1. 掌握 Android Studio、Jupyter Notebook 及 Visual Studio Code 等开发环境的安装与配置方法。
2. 熟悉 Anaconda 的 Python 环境管理方式，能够启动并使用 Jupyter Notebook。
3. 能够在 VS Code 中配置 Python 与 Jupyter 插件，创建并运行交互式笔记。
4. 将安装过程整理为 Markdown 文档并上传至 GitHub/Gitee，培养文档撰写与版本管理习惯。

## 二、实验环境

- 操作系统：Windows 11（64-bit）
- 所需软件：
  - Android Studio (2024.1.1 Koala 或更高版本)
  - Anaconda3（包含 Jupyter Notebook）
  - Visual Studio Code
- 网络环境：需连接互联网以下载安装包与依赖

## 三、实验内容与步骤

### 3.1 安装 Android Studio

1. **下载安装**

   - 访问 [Android Studio 官网](https://developer.android.com/studio) 下载最新版（版本号 ≥ 4.1，实际下载 2024.1.1）。
   - 运行安装程序，按默认设置完成安装。选择组件时确保勾选 Android Virtual Device。

2. **配置国内镜像（阿里云 Maven）**

   - 启动 Android Studio，进入 `Customize > All settings…`。

   - 搜索 `HTTP Proxy`，设置代理为 `No proxy`（若无需代理）。

   - 配置 Gradle 仓库镜像：在项目级 `build.gradle`（或 `settings.gradle.kts`）中添加阿里云镜像：

     kotlin

     ```
     repositories {
         maven { url 'https://maven.aliyun.com/repository/public' }
         maven { url 'https://maven.aliyun.com/repository/google' }
         google()
         mavenCentral()
     }
     ```

     

3. **新建项目并编译运行**

   - 选择 “New Project” → “Empty Views Activity”，按向导完成创建。
   - 首次编译时 Gradle 会自动下载依赖包，等待构建完成。
   - 点击 “Run” 按钮，选择已创建的虚拟设备（如 Pixel 6 API 34）或连接真机，运行默认应用。
   - 确认模拟器中可正常显示 “Hello World” 界面。

### 3.2 安装 Jupyter Notebook（通过 Anaconda）

1. **下载安装 Anaconda**

   - 访问 [Anaconda 官网](https://www.anaconda.com/download) 获取 Windows 安装包。
   - 运行安装程序，注意：
     - 安装路径不可包含中文或空格。
     - 选择 “Just Me” 安装方式。
     - 勾选 “Add Anaconda to my PATH environment variable”（可选，或后续手动启动 Anaconda Prompt）。
   - 安装完成后，从开始菜单打开 **Anaconda Navigator**，成功启动表示安装完成。
   - 使用 **Anaconda Prompt** 执行 `conda list` 验证已安装包。

2. **启动 Jupyter Notebook**

   - 方式一：在 Anaconda Navigator 中点击 “Jupyter Notebook” 启动。
   - 方式二：在 Anaconda Prompt 中输入 `jupyter notebook`。
   - 默认打开浏览器，页面显示用户主目录的文件列表。

3. **更改 Notebook 默认目录（可选）**

   - 在命令行中生成配置文件：`jupyter notebook --generate-config`
   - 找到 `jupyter_notebook_config.py`，修改 `c.NotebookApp.notebook_dir = 'D:/MyNotebooks'`（路径自定）。
   - 重新启动 Jupyter Notebook。

4. **新建并运行一个 Notebook**

   - 点击 “New” → “Python 3”，创建一个新笔记。

   - 在第一个单元格中输入：

     python

     ```
     import sys
     print("Python version:", sys.version)
     print("Hello, Jupyter!")
     ```

     

   - 按 `Shift+Enter` 执行，观察输出结果。

### 3.3 安装 Visual Studio Code 并配置 Python/Jupyter 插件

1. **下载安装 VS Code**
   - 前往 [VS Code 官网](https://code.visualstudio.com/) 下载稳定版，双击安装。
   - 安装时可勾选 “添加到 PATH” 等实用选项。
2. **安装必备扩展**
   - 启动 VS Code，点击左侧活动栏的扩展图标（Ctrl+Shift+X）。
   - 搜索并安装以下扩展：
     - **Python**（Microsoft）
     - **Jupyter**（Microsoft）
     - **Jupyter Keymap**（Microsoft，可选）
3. **创建并运行 Jupyter Notebook**
   - 按 `Ctrl+Shift+P` 调出命令面板，输入 `Create: New Jupyter Notebook`。
   - 选择 Python 解释器（Anaconda 中的 Python）。
   - 在第一个单元格输入测试代码并执行，验证环境可用。
4. **关联 Anaconda 环境**
   - 若 VS Code 未自动识别 Anaconda Python，可在设置中搜索 `Python: Default Interpreter Path`，指向 Anaconda 安装目录下的 `python.exe`。
   - 抑或在命令面板中运行 `Python: Select Interpreter` 选择对应环境。

## 五、问题与解决方案

1. **Android Studio 首次编译卡住**
   - 原因：Gradle 依赖下载缓慢。
   - 解决：在 `settings.gradle.kts` 中添加阿里云镜像地址，并关闭代理设置，重新同步。
2. **Jupyter Notebook 启动后浏览器空白**
   - 原因：防火墙或杀毒软件拦截本地端口。
   - 解决：暂时关闭防火墙或将 Jupyter 使用的端口（默认8888）加入白名单。
3. **VS Code 中无法选择 Anaconda Python 解释器**
   - 原因：未正确安装 Python 扩展，或解释器路径未被扫描到。
   - 解决：在命令面板执行 `Python: Select Interpreter`，若列表为空，手动输入 Anaconda 安装路径下的 `python.exe`。
