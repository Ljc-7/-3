**任务一：创建首个 Kotlin 应用**
目标：创建一个基础的 Kotlin + Jetpack Compose 项目，并成功运行到设备上。
详细步骤：

1. 打开 Android Studio，选择 “New Project” → “Empty Activity”。
2. 应用名称填写 “MyFirstKotlinApp”，包名建议为 com.example.myfirstkotlinapp，语言选择 Kotlin，Minimum SDK 选择 API 21（Android 5.0），这是 Jetpack Compose 支持的最低版本。
3. 点击 Finish，等待 Gradle 同步和项目构建完成（首次同步可能需要下载依赖，耗时约 2-5 分钟）。
4. 创建或启动 Android 模拟器（推荐使用 Pixel 4 API 30 镜像），或者用 USB 线连接真机并开启开发者选项和 USB 调试。
5. 点击运行按钮（绿色三角形），选择目标设备，观察应用成功安装并启动，屏幕显示 “Hello, Android!” 或默认文本。

检查点：

- 项目中的 MainActivity.kt 文件应包含 setContent 块，且使用了 Compose 函数。
- 如果运行失败，可检查 build.gradle 中的 compose 版本是否与 Kotlin 版本兼容。

**任务二：实践 Compose 布局**
目标：学习 Jetpack Compose 的基础组件，并构建一个可交互的简单界面。
详细步骤：

1. 打开 app/src/main/java/.../MainActivity.kt，找到 setContent 代码块内的 Greeting 函数。
2. 修改 Greeting 函数，使用 Column 垂直布局，内部包含一个 Text 文本和一个 Button 按钮。设置 Column 的对齐方式为水平居中和顶部开始。
3. 引入状态管理：使用 var message by remember { mutableStateOf("欢迎使用 Compose") } 定义一个可变状态。
4. 在 Text 中显示 message 的值，在 Button 的 onClick 回调中将 message 修改为 “按钮已点击！第 X 次”。（可以使用计数器记录点击次数）。
5. 可选：增加一个 Spacer 或 padding 让布局更美观，修改字体大小和按钮颜色。
6. 运行应用，点击按钮验证文本是否变化。

核心代码示例：
@Composable
fun GreetingScreen() {
var clickCount by remember { mutableStateOf(0) }
val displayText = if (clickCount == 0) "欢迎使用 Compose" else "按钮已点击 $clickCount 次"
Column(
modifier = Modifier.fillMaxSize().padding(16.dp),
horizontalAlignment = Alignment.CenterHorizontally,
verticalArrangement = Arrangement.Center
) {
Text(text = displayText, fontSize = 24.sp, color = Color.Blue)
Spacer(modifier = Modifier.height(16.dp))
Button(onClick = { clickCount++ }) {
Text("点我增加次数")
}
}
}

**任务三：完成面向 AI 应用的 Compose 布局**
目标：设计一个 AI 识别应用的主界面，满足实验指导书中的布局要求。
界面要求（根据实验指导书）：

- 顶部栏：显示应用标题（如 “AI 智能相机”）及操作入口（如设置图标或更多选项）。
- 预览区：占位区域，后续替换为 CameraX 相机预览（当前可用 Box 加背景色和占位文字模拟）。
- 结果区：显示模型名称、识别结果、置信度、推理时间等信息，使用 Card + Column 展现，建议每条信息单独一行。
- 按钮区：包含“拍照识别”、“相册导入”、“切换模型”、“清空结果”四个按钮，合理排列（可用 Row 或 FlowRow）。

实现要点与建议：

- 最外层使用 Column，设置 Modifier.fillMaxSize() 和适当的 padding（如 12.dp）。
- 顶部栏：可以使用 TopAppBar 组件，其中 title 设置为 Text("AI 智能相机")，actions 可以添加一个 IconButton。
- 预览区：使用 Box 设置固定高度（建议 280dp），背景色为 Color(0xFFE0E0E0)，内部居中放置 Text("相机预览区")。
- 结果区：使用 Card，设置 elevation = 4.dp 和 shape = RoundedCornerShape(8.dp)。内部 Column 设置 padding 16.dp，每个信息用 Row 或 Text 展示，例如：Text("模型：MobileNetV3")，Text("识别结果：猫")，Text("置信度：0.94")，Text("推理时间：23ms")。
- 按钮区：使用 Row 设置 horizontalArrangement = Arrangement.SpaceEvenly，每个 Button 的 modifier 可设置为 Modifier.weight(1f) 等宽分配，同时按钮文字可能折行，建议使用较小字号或简短文字（如“拍照”、“相册”、“模型”、“清空”）。
- 增加必要的 Spacer 分隔各部分，使布局不拥挤。

完整代码结构示意（核心部分可粘贴）：
@Composable
fun AiCameraScreen() {
Column(modifier = Modifier.fillMaxSize().padding(16.dp)) {
// 顶部栏
TopAppBar(
title = { Text("AI 智能相机") },
actions = {
IconButton(onClick = { /* 设置操作 */ }) {
Icon(Icons.Default.Settings, contentDescription = "设置")
}
}
)
Spacer(modifier = Modifier.height(8.dp))
// 预览区占位
Box(
modifier = Modifier.fillMaxWidth().height(280.dp).background(Color.LightGray),
contentAlignment = Alignment.Center
) {
Text("相机预览区", color = Color.DarkGray)
}
Spacer(modifier = Modifier.height(16.dp))
// 结果区
Card(modifier = Modifier.fillMaxWidth(), elevation = 4.dp) {
Column(modifier = Modifier.padding(16.dp)) {
Text(text = "模型：MobileNetV3", fontSize = 16.sp)
Text(text = "识别结果：猫", fontSize = 16.sp, color = Color.Blue)
Text(text = "置信度：0.94", fontSize = 14.sp)
Text(text = "推理时间：23ms", fontSize = 14.sp)
}
}
Spacer(modifier = Modifier.height(16.dp))
// 按钮区
Row(horizontalArrangement = Arrangement.SpaceEvenly) {
Button(onClick = {}) { Text("拍照识别") }
Button(onClick = {}) { Text("相册导入") }
Button(onClick = {}) { Text("切换模型") }
Button(onClick = {}) { Text("清空结果") }
}
}
}

注意事项：

- 如果预览区过高导致按钮被挤出屏幕，可以将预览区的 Modifier.weight(1f) 与 Column 的 verticalArrangement = Arrangement.SpaceBetween 结合使用。
- 为了美观，可添加阴影、圆角和适当的间距。

**实验总结**：
通过本次实验，我深入体验了 Kotlin + Jetpack Compose 开发 Android 应用的完整流程。相较于传统的 XML 布局和 findViewById 方式，Compose 声明式 UI 极大简化了界面代码的编写，状态管理也更加直观。实验中我重点练习了以下知识点：

- Kotlin 基础：lambda 表达式作为 onClick 回调，mutableStateOf 委托属性实现响应式状态。
- Compose 布局：Column、Row、Box 的嵌套使用，Modifier 的链式调用（fillMaxSize、padding、background 等）。
- 组件定制：TopAppBar、Card、Button、Text、Icon 的常用属性。
- 代码组织：将不同部分的 UI 拆分为独立的 @Composable 函数，提高可读性和复用性。

在任务三的设计过程中，我遇到了一个小问题：预览区使用固定高度后，按钮区在某些小屏幕设备上会超出屏幕底部。解决方法是将外层 Column 的 verticalArrangement 设置为 SpaceBetween，并为预览区添加 Modifier.weight(1f)，这样预览区会占用剩余空间，按钮区自动靠底。此外，为了按钮区适配不同宽度的屏幕，我使用了 Row 配合 weight 让每个按钮等宽，并设置了按钮文字为两行以内可显示。

本次实验为后续开发完整的 AI 图像识别应用打下了扎实的基础。下一步计划：将预览区的占位 Box 替换为 CameraX 的 PreviewView，并为按钮添加实际功能（调用相机拍照、从相册选取、调用识别模型、清空结果显示）。同时，我会进一步学习 Kotlin 协程和 Flow，用于处理相机帧和模型推理的异步任务。
