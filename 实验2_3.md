**四、实验内容与步骤**

本实验按照教程分步完成，包含基础功能和可选扩展。

**任务概览**：

- 创建一个 Empty Activity 项目，语言选择 Kotlin，Minimum SDK API 21。
- 在 build.gradle 中添加 CameraX 相关依赖。
- 使用 XML 布局文件设计界面（包含 PreviewView、按钮等）。
- 申请相机和存储权限（运行时权限）。
- 实现 Preview（预览）、ImageCapture（拍照）、VideoCapture（录像）功能。
- （扩展）尝试集成 ImageAnalysis，对每一帧图像进行处理（如灰度化或边缘检测）。

**详细步骤**：

**步骤1：创建项目并添加依赖**

1. 在 Android Studio 中新建 Empty Activity，命名例如 “CameraXApp”，包名自定义，语言 Kotlin，Minimum SDK API 21。

2. 打开 app/build.gradle.kts（或 build.gradle），在 dependencies 块中添加以下内容（教程中使用的版本为 1.5.0-alpha06，可替换为最新稳定版）：

   text

   ```
   val camerax_version = "1.5.0-alpha06"  
   implementation("androidx.camera:camera-core:${camerax_version}")  
   implementation("androidx.camera:camera-camera2:${camerax_version}")  
   implementation("androidx.camera:camera-lifecycle:${camerax_version}")  
   implementation("androidx.camera:camera-video:${camerax_version}")  
   implementation("androidx.camera:camera-view:${camerax_version}")  
   // 可选：ML Kit Vision 集成  
   // implementation("androidx.camera:camera-mlkit-vision:${camerax_version}")  
   // 可选：Extensions 库  
   // implementation("androidx.camera:camera-extensions:${camerax_version}")  
   ```

   

3. 同步 Gradle。

**步骤2：设计布局（XML）**
创建或修改 `activity_main.xml`，包含：

- 一个 `androidx.camera.view.PreviewView`，用于显示相机预览，占据大部分屏幕。
- 三个按钮：拍照、开始录像、停止录像（或一个切换录像的按钮）。
- 可选：一个 TextView 用于显示状态信息。

示例布局结构（可省略具体代码，仅描述）：
顶部为 PreviewView（layout_width=match_parent，layout_height=0dp，weight=1）。
底部为 LinearLayout 或 Row 放置三个按钮，垂直或水平排列。

**步骤3：申请运行时权限**
在 AndroidManifest.xml 中添加：

text

```
<uses-permission android:name="android.permission.CAMERA" />  
<uses-permission android:name="android.permission.RECORD_AUDIO" />   <!-- 录像需要音频权限 -->  
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />  
```



在 MainActivity 中，检查并请求权限（使用 ActivityResultContracts）。

**步骤4：实现预览（Preview）**
在 `onCreate` 中或单独函数内：

- 获取 CameraProvider（使用 ProcessCameraProvider.getInstance）。
- 创建 Preview 用例，并将其与 PreviewView 的 surfaceProvider 绑定。
- 选择后置摄像头（CameraSelector.DEFAULT_BACK_CAMERA）。
- 将 Preview 绑定到生命周期（使用 cameraProvider.bindToLifecycle）。

**步骤5：实现拍照（ImageCapture）**

- 创建 ImageCapture 用例，配置输出目录和文件命名。
- 在拍照按钮的点击事件中，调用 imageCapture.takePicture，使用 OnImageSavedCallback 处理保存成功或失败。
- 图片保存到应用的媒体目录或外部存储，并可以显示 Toast 提示。

**步骤6：实现录像（VideoCapture）**

- 创建 VideoCapture 用例（注意需要同时绑定 Preview 和 VideoCapture，且 VideoCapture 需要录音权限）。
- 设置输出文件路径（File）。
- 开始录像：调用 videoCapture.startRecording。
- 停止录像：调用 videoCapture.stopRecording。
- 注意：VideoCapture 需要与 Preview 绑定在同一生命周期，且需要处理音频权限。

**步骤7：运行测试**
在真机上运行，验证：

- 预览画面实时显示。
- 点击拍照可以保存照片到相册。
- 点击录像可以录制视频并保存。

**步骤8（扩展实验）：集成 ImageAnalysis**

- 添加依赖：implementation("androidx.camera:camera-mlkit-vision:${camerax_version}") 或单独添加 analysis 库。
- 创建 ImageAnalysis 用例，设置分析器（实现 ImageAnalysis.Analyzer 接口）。
- 在分析器中对每一帧图像进行处理（例如获取像素数据，计算亮度平均值或进行简单的边缘检测）。
- 将 ImageAnalysis 与其他用例一起绑定（注意性能，设置 backpressure 策略为 STRATEGY_KEEP_ONLY_LATEST）。
- 可选：在界面上显示分析结果（如帧率、亮度值）。

**注意事项**：

- CameraX 需要在主线程绑定，但图像分析的回调在非主线程，更新 UI 需切换线程。
- 录像功能需要 API 21 以上，且部分设备可能不支持 VideoCapture，建议做好异常处理。
- 记得在不需要相机时解除绑定（自动由 lifecycle 管理，但也可手动 unbind）。

▶ 截图插入位置 1：放置应用运行时的主界面截图，应显示预览画面（例如对着某个物体）以及拍照、录像按钮。

▶ 截图插入位置 2：放置拍照后保存的照片在手机相册中显示的截图（或拍照成功时的 Toast 弹窗）。

▶ 截图插入位置 3（可选扩展实验）：放置 ImageAnalysis 实时分析结果（如亮度数值）显示在界面的截图。

**五、实验结果与总结**

**实验结果**：

1. 成功实现了 CameraX 的预览功能，预览画面清晰流畅，无卡顿。
2. 拍照功能正常，点击拍照按钮后能保存高质量图片到设备相册，并通过 Toast 提示用户保存路径。
3. 录像功能正常，可以录制带有音频的视频文件，停止录像后视频可在相册中播放。
4. 扩展实验（ImageAnalysis）完成了对每一帧图像的亮度分析，并在界面上实时显示平均亮度值，为后续集成 AI 模型推理打下基础。
5. 代码已上传至 GitHub 仓库，包含完整的项目源码、README 文档以及必要的截图。

**实验总结**：
本次实验让我深刻理解了 CameraX 作为现代化相机开发库的优势。相比于直接使用 Camera2 API 需要编写大量底层代码，CameraX 通过用例（UseCase）的方式抽象了常见的相机操作（预览、拍照、录像、分析），大大简化了开发流程。

关键收获包括：

- **生命周期绑定**：使用 `ProcessCameraProvider` 和 `lifecycleOwner`，相机资源能自动跟随 Activity 生命周期管理，避免了手动释放的麻烦。
- **权限处理**：运行时权限请求与 ActivityResult API 的结合，使代码更清晰。
- **录像与预览同时进行**：通过 `bindToLifecycle` 一次绑定多个用例，CameraX 会自动协调它们的资源分配。
- **图像分析**：`ImageAnalysis` 用例提供了访问每一帧的能力，通过自定义分析器可以实现实时处理，但要注意性能优化，避免丢帧。

遇到的困难及解决方法：

- **录像无声**：起初忘记添加 `RECORD_AUDIO` 权限，添加后正常。
- **录像保存路径问题**：Android 10 以上分区存储限制，需要使用 MediaStore API 或 getExternalFilesDir。解决方案：使用 `File` 指向 `getExternalFilesDir` 下的 Movies 目录。
- **ImageAnalysis 更新 UI 卡顿**：分析器回调在子线程，直接更新 UI 会抛异常。使用 `Handler(Looper.getMainLooper())` 切换到主线程。

本次实验为后续开发基于相机预览的智能应用（如物体识别、文字识别、人脸检测）奠定了坚实的基础。下一步计划：将 ImageAnalysis 与 ML Kit 或自定义 TensorFlow Lite 模型集成，实现实时分类或检测。
