## 四、关键代码示例与解析

以下是从练习中摘录的典型题目及代码。

### 4.1 空安全与安全调用

kotlin

```
fun sendMessageToClient(client: Client?, message: String, emailer: Emailer) {
    client?.personalInfo?.email?.let { emailer.sendMessage(it, message) }
}
```



**解析**：

- `?.` 安全调用操作符确保只有当左侧对象非空时才访问右侧成员。
- `let` 函数在非空时执行代码块，避免了显式的 `if (xxx != null)` 判断。
- 整个链式调用优雅地处理了多级可能为空的属性。

### 4.2 集合操作：filter 与 map

kotlin

```
fun getCitiesWithMoreThan1000People(cities: List<City>): List<City> {
    return cities.filter { it.population > 1000 }
}
```



**解析**：

- `filter` 接收一个 Lambda 条件，返回满足条件的元素组成的新列表。
- 比传统的 `for` 循环更加简洁、可读性更强，属于函数式编程风格。

### 4.3 数据类与解构声明

kotlin

```
data class Person(val name: String, val age: Int)

fun getOldestPerson(people: List<Person>): Person? {
    return people.maxByOrNull { it.age }
}
```



**解析**：

- `data class` 自动生成 `equals`、`hashCode`、`toString` 和 `copy` 方法，非常适合作为数据模型。
- `maxByOrNull` 安全地处理空集合，返回可空类型。

### 4.4 when 表达式

kotlin

```
fun describe(obj: Any): String = when (obj) {
    1          -> "One"
    "Hello"    -> "Greeting"
    is Long    -> "Long number"
    !is String -> "Not a string"
    else       -> "Unknown"
}
```



**解析**：

- `when` 可以匹配值、类型、区间、布尔条件，比 Java 的 `switch` 强大得多。
- 可作为表达式直接返回值，使代码非常紧凑。

## 六、遇到的困难与解决方法

### 6.1 flatMap 的理解

**问题**：在 `Flat collection` 题目中，需要将列表的列表展平并转换。最初使用 `map` + `flatten`，代码冗余且不易读。
**解决**：查阅官方文档后，理解 `flatMap` 就是 `map` 后接 `flatten`，直接使用 `list.flatMap { it.items }` 一行代码解决。

### 6.2 lateinit 与 lazy 的区别

**问题**：不清楚何时使用 `lateinit`，何时使用 `by lazy`。
**解决**：通过资料学习：

- `lateinit` 用于可变属性（`var`），需要手动在适当位置初始化，常用于依赖注入或 Android 的 `onCreate`。
- `lazy` 用于只读属性（`val`），委托给 Lambda，第一次访问时自动初始化，线程安全。

### 6.3 when 表达式忘记 else 分支

**问题**：在写 `when` 表达式时，未覆盖所有可能情况，导致编译错误。
**解决**：除非使用枚举或密封类（sealed class）且已穷举所有分支，否则始终添加 `else` 分支作为默认处理。

## 七、实验总结

通过本次 Kotlin Koans 实验，我系统掌握了 Kotlin 的核心语法和惯用法，主要收获如下：

1. **空安全**：Kotlin 的类型系统强制区分可空与非空，配合 `?.`、`?:`、`let` 等操作，彻底告别了 Java 中常见的 `NullPointerException`。
2. **函数式集合操作**：`filter`、`map`、`reduce` 等操作让数据处理代码非常简洁，显著提升了可读性和开发效率，减少了循环和可变状态。
3. **数据类**：只需一行 `data class` 即可替代 Java 中大量的 `getter`、`setter`、`equals`、`hashCode` 和 `toString`，非常适合 Android 中的 UI 状态、网络响应、数据库实体等场景。
4. **表达式风格**：`if`、`when`、函数体都可以作为表达式返回值，使代码更加紧凑、富有表现力。
5. **作用域函数**：`let`、`apply`、`with`、`run` 等提供了灵活的上下文切换能力，在对象初始化和链式调用中非常实用。
